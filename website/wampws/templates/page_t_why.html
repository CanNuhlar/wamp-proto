The WebSocket Protocol is already built into modern browsers and provides full-duplex, low-latency communication. However, as such, it is quite low-level. Web apps often have a need for higher level messaging patterns such as Publish & Subscribe (PubSub) or Remote Procedure Calls (RPC). This is where The WebSocket Application Messaging Protocol enters.

WAMP runs on top of raw WebSocket and provides PubSub and RPC. Technically, WAMP is a proper WebSocket subprotocol that uses JSON as message serialization format. WAMP was designed to be easy to use and simple to implement.
